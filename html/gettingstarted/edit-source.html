Edit Clojure source code

By Creating a New Clojure File

Clojure files must be located in normal Java source directories. If you place files with the .clj extension outside of a Java source directory, Eclipse will not see it on the classpath of the project, and it will not be possible to load it via the REPL or to select it as an automatically loaded file in the launcher customization wizard.

  • You can use File > New > Clojure File to create a Clojure file
  • Inside the Java source directory, follow the classical Clojure conventions to place your file, according to the namespace it belongs to. (e.g. you want to create namespace 'tests.clojure.first, then create Java package "tests.clojure", and create Clojure file first.clj in it.

By Importing Existing Code

Do as you would in a classic Java project. Just remember to place all Clojure files in a Java source directory.

What Features Does The Editor Offer?

Syntax Highlighting

  • Rainbow parenthesis (different colors for different nesting levels)
  • Highlights in error closing parens that have no matching opening parens

Code completion

Currently, does what is called "Hippie completion" by Eclipse: code completion for all available words on all open files.

Errors reporting

In conjunction with auto-compile functionality, compilation problems are reported as problem markers:

  • A summary of the problems is displayed in the Problems view
  • Files that have problems are marked in the Package Explorer
  • Problems are marked at the correct line in the Clojure editor
  • A double-click on a problem in the Problems view jumps to the correct line in the Clojure editor

Interaction with a launched REPL

The Clojure Plugin tries to follow as much as possible classical "Slime" conventions. It also tries to correctly integrate with Eclipse shortcuts where Slime functionality is semantically equivalent to an existing Eclipse functionality.

Name Keyboard Shortcuts Description
beginning-of-defn C-M-a or C-S-A Move to beginning of current or preceding top-level s-expression (not just defns)
end-of-defn C-M-e or C-S-E Move to end of current or preceding top-level s-expression (not just defns)
mark-defn C-M-h Select current or following top-level s-expression
eval-defn + eval-region C-M-x If a non-empty text is selected, evaluate it, or, if no text is selected, evaluate the top-level s-expression
load file C-M-l Loads the saved version of the currently edited file into the REPL (via Clojure"s "load-file" function)

Remarks:

  • The Windows keys combination Ctrl+Alt can also be replaced by the single AltGr key
  • C stands for Ctrl key, M stands for Meta key (Alt key on Windows), S stands for Shift
